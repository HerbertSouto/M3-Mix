# Overview

M3-Mix is a web application that brings Marketing Mix Modeling (MMM) to the day-to-day of the marketing manager — no statistics background required.

## What the application delivers

| Output | Description |
|--------|-------------|
| **ROAS by channel** | Return on ad spend calculated from real contribution, free from last-click bias |
| **Revenue contribution** | Decomposition of how much each channel and the baseline explain of total revenue |
| **Saturation curves** | Marginal return per channel — identifies where more investment stops paying off |
| **Adstock decay** | Residual media effect modeled week by week |
| **Executive narrative** | Plain-language report generated by an LLM |
| **Budget optimization** | Cross-channel reallocation simulation for a given total budget |

## Stack

| Layer | Technology |
|-------|------------|
| Frontend | Next.js 16 (App Router) + TypeScript |
| UI | Tailwind CSS + shadcn/ui |
| Charts | Recharts |
| AI — narrative & chat | LLaMA 3.3 70B via Groq |
| Backend | FastAPI (Python 3.11+) |
| MMM | PyMC-Marketing 0.x + PyMC |
| Optimization | scipy.optimize (SLSQP) |
| Database | Supabase (PostgreSQL + Storage) |
| Frontend deploy | Vercel |
| Backend deploy | Railway |

## Flow summary

```
User uploads CSV
      ↓
Next.js /api/upload
  → validates file (type, size, rate limit by IP)
  → uploads to Supabase Storage
  → creates record in analyses (status: processing)
  → fires POST /analyze to FastAPI
      ↓
FastAPI (background task)
  → downloads the CSV
  → fits MMM model with PyMC-Marketing
  → extracts ROAS, contributions, saturation, adstock
  → generates narrative with Groq
  → saves results to Supabase
      ↓
Frontend polling /api/analysis/[id]/status
  → when completed, renders the report
```
